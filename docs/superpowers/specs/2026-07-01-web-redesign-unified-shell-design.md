# C5 网页后台重设计（统一外壳 · Linear 风 · 地图为家）

> 状态：设计稿（待评审 → 待实现）｜日期：2026-07-01｜范围：`web-admin` 前端（**后端零改动**）
> 前序：[地图主页设计](./2026-06-30-map-first-home-design.md)、[前后端分离部署](./2026-06-29-frontend-backend-split-deploy-design.md)

## 0. 背景与动机

「地图主页」以「新增地图页、保留现有页面」的增量方式落地后，后台产生了信息架构债：

1. **两个首页并存**：`/` 地图总览（新）+ `/dashboard` 工作台（旧仪表盘），职责重叠。
2. **两套导航范式并存**：地图主页是「全屏地图 + 悬浮菜单」（无侧栏）；其它页是「AntD Pro 侧栏 + 表格」。切换时割裂。
3. **视觉风格不一致**：地图页毛玻璃悬浮风 vs 其它页 AntD 默认风。
4. **菜单 10 项平铺**，把视图 / 业务实体 / 系统配置混在一起，无分组。

本设计把整站收敛为**一套统一外壳 + 一套设计语言**，用户流程即导航结构。**逻辑、接口、数据流全部不变，仅改外观与归位。**

## 1. 已确认决策

| # | 决策 |
|---|---|
| 1 | **统一外壳 · 地图为沉浸首页**：全站一套顶栏 + 侧栏 + 设计系统；首页是大地图（填内容区），其它页在同一外壳内换成表格/表单。 |
| 2 | **视觉风格：现代极简·亮色（Linear/Vercel 风）**：亮底 + 中性灰阶 + 单一强调色（靛蓝）；1px 细边框替代重阴影；紧凑间距；去 AntD 默认味。 |
| 3 | **删除独立 `/dashboard`**：首页 `/` 即地图 + 顶部紧凑指标条；深度看板仍是 `/stats`。一个家。 |
| 4 | **侧栏按流程四分组**：概览 / 业务 / 分析 / 系统。 |
| 5 | **`⌘K` 命令面板放 P3**（非首期）。 |
| 6 | **桌面优先**；平板/窄屏响应式放 P3。 |

（#3/#4/#5/#6 为推荐默认，评审可否决。）

## 2. 设计系统（根治风格不一致）

一套 token，**AntD 组件 + 地图悬浮层 + 自定义页面全部继承**，从源头统一。落地在 `styles/theme.ts`（AntD `ConfigProvider` theme tokens）+ `styles/tokens.css`（CSS 变量），一处改、全站变。

- **颜色**：底 `#ffffff` / `#fafafa`；文本 `#111827`→`#6b7280`（三档灰阶）；边框 `#e8e8ec`（1px）；强调靛蓝 `#635bff`；语义色低饱和（success/warning/danger）。
- **表面**：卡片 = 1px 边框 + 极弱阴影；圆角统一 `8px`；阴影只留给弹层/悬浮层/命令面板。
- **排版**：Inter；标题/正文/次要三档；等宽数字（里程/时长/计数）。
- **密度**：表格「紧凑舒适」，统一行高与筛选栏；间距走 4/8 栅格。
- **AntD 主题**：调 `token.colorPrimary/colorBorder/borderRadius/controlHeight/boxShadow` 等，使 Table/Form/Modal/Tag 自动落到 Linear 风，避免逐组件改。

## 3. 统一外壳 AppShell

替换现有 `AppLayout`（AntD Pro `ProLayout`）为自有轻量外壳（或深度定制 ProLayout），所有受保护路由（含地图首页）都在其内。

```
┌───────────────────────────────────────────────┐
│ C5 · 面包屑/页名        ⌘K 搜索(P3)        👤 ▾ │  顶栏(全站一致)
├──────┬────────────────────────────────────────┤
│ 概览 │                                        │
│  地图│                                        │
│ 业务 │        内容区(每页在此渲染)             │
│  项目│   · 首页 = 沉浸大地图(侧栏可收窄)        │
│  巡查│   · 其它页 = 表格 / 表单 / 详情          │
│  问题│                                        │
│  全景│                                        │
│ 分析 │                                        │
│  统计│                                        │
│  导出│                                        │
│ 系统 │                                        │
│  字典│                                        │
│  权限│                                        │
└──────┴────────────────────────────────────────┘
```

- **左侧栏**：可收起；按流程分组（§4）；当前项高亮；图标 + 文案；权限过滤（复用 `permissionCodes`）。
- **顶栏**：面包屑/页名 + 全局搜索（`⌘K`，P3）+ 用户菜单（改密/登出，复用 `useLogout` + `ChangePasswordModal`）。
- **内容区**：每页渲染处。首页地图**填满内容区**（进首页时侧栏可自动收窄以沉浸），但顶栏与设计语言与全站一致——**地图不再是独立全屏异类**。地图上的图层开关/选中卡/指标条均用同一套 token。

组件拆分（高内聚小文件）：`AppShell` / `AppSidebar`（分组渲染）/ `AppTopBar` / `CommandPalette`(P3)。

## 4. 使用流程 = 导航结构

侧栏按真实工作流分组，导航即流程（准备 → 采集 → 审核 → 分析）：

| 分组 | 菜单项 | 路由 | 流程阶段 |
|---|---|---|---|
| **概览** | 地图（首页） | `/` | 空间视角总览 |
| **业务** | 项目 · 巡查 · 问题 · 全景 | `/projects` `/inspections` `/problems` `/panorama` | 准备 → 审核 |
| **分析** | 统计 · 导出 | `/stats` `/exports` | 分析 → 输出 |
| **系统** | 字典 · 权限 | `/dict` `/rbac` | 准备（配置） |

**收敛**：删除 `/dashboard`（旧仪表盘）；首页 `/` = 地图 + 顶部紧凑指标条（巡查数/问题数/里程等，取 `/stats/overview`）；深度看板仍是 `/stats`。→ 消除「两个首页」。

数据流水线（页面据此各就各位）：

```
① 准备(管理员)        ② 采集(现场APP·待建)    ③ 审核(管理员)          ④ 分析/输出
权限·字典·项目/任务  → 连相机·巡查·轨迹·全景 → 巡查·问题·全景·地图   → 统计·导出
                       ·标问题·上传
```

## 5. 页面逐个改造（换皮 + 归位，逻辑不动）

| 页面 | 改造 |
|---|---|
| 地图首页 `/` | 从「全屏异类」收进 AppShell；加顶部指标条；图层/选中卡沿用同一 token；删 `/dashboard` 后作为唯一首页 |
| 项目/巡查/问题 列表 | 统一 Linear 风数据表（1px 边框、紧凑、行悬停、统一筛选栏 + 分页） |
| 各详情页 | 统一「详情页」版式（左信息 + 右关联/时间线） |
| 全景 | 统一查看器外壳；入口来自问题/地图点位 |
| 统计 | 指标卡 + 图表统一到新 token |
| 导出/字典/权限 | 统一表格/表单/弹窗风格 |

**不变**：所有 hooks / API / 数据流 / 权限门控 / 坐标转换（`toLatLng`）/ 现有业务逻辑与测试意图。

## 6. 分阶段实施（增量，每期可单独上线，测试全绿）

- **P0 地基**：设计 token（theme.ts + tokens.css）+ AppShell（分组侧栏 + 顶栏）+ 地图首页收进外壳 + 删 `/dashboard` + 首页指标条。→ 一次性消除「两范式/两首页/风格不一」。
- **P1 列表/详情换皮**：项目 · 巡查 · 问题 的表格与详情。
- **P2 分析/配置换皮**：统计 · 导出 · 字典 · 权限。
- **P3 增强**：`⌘K` 命令面板、响应式（平板/窄屏）、空态/加载态精修。

## 7. 风险与权衡

- **换皮范围大**：但走 AntD token，单点改动放大到全站，性价比高；分期降风险。
- **AppShell 替换 ProLayout**：需迁移动态菜单/面包屑/用户菜单/权限过滤（现均已存在，平移即可）。
- **地图首页收进外壳**：需重排层叠（保留已修的 isolation/stacking 经验，见地图主页 spec）。
- 每期跑通 `typecheck` / `eslint` / 全量 vitest，保持绿；新增外壳组件补测试。

## 8. 验收标准

- 全站（含地图首页）共用同一外壳（顶栏 + 分组侧栏）与同一套设计 token，视觉一致。
- `/` 即地图 + 指标条；无独立 `/dashboard`；侧栏按 概览/业务/分析/系统 分组且权限过滤生效。
- 所有页面逻辑/接口/数据不变；现有测试意图保留，全量测试通过。
- 地图首页在外壳内层叠正确（悬浮控件可点击，地图可交互）。
- 坐标转换仍仅经 `toLatLng`；无新增外域（CSP 不放宽）。
